Vulnerable Upload Scenarios (Testing Only)

These endpoints demonstrate specific CWE-434 vulnerabilities

Scenario 1: No Protection

Vulnerability: CWE-434

This endpoint has NO security controls:

  • No file type validation
  • No content scanning
  • No size limits
  • Original filename preserved
⚠️ Any file type accepted!
Scenario 2: Weak Protection

Vulnerability: CWE-434

Weak security controls (bypassable):

  • Header-based size check (bypassable)
  • Extension blacklist (incomplete)
  • No MIME type verification
  • No content scanning
⚠️ Blacklist: .exe, .sh, .bat, .php
Scenario 3: Encrypted Bypass

Vulnerability: CWE-434

Encryption bypass vulnerability:

  • Files encrypted with AES-256
  • Scanned in encrypted form only
  • Decrypted later without re-scan
  • Malicious content hidden in encryption
⚠️ Encrypted files bypass scanning
